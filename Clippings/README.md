# Clippings

Obsidian Web Clipper 和文章/网页剪藏的默认目录。

规则：

- 剪藏文件由 Obsidian Web Clipper 插件生成，格式以插件配置为准。
- 不维护单独的 DailyVault clipping template。
- 原文和剪藏放这里，不按日期塞进 `Daily/`。
- Web Clipper 只负责把网页原文直接写入 `Clippings/`；它不会自动关联 Daily 或 Sources。
- `Daily/YYYYMMDD.md` 如果需要引用剪藏，只写一个可读链接和自己的判断，例如 `[[Clippings/YYYYMMDD-title|文章标题]]：为什么今天有用`。
- 只有当剪藏值得分类、评分、打 tag、进入年度回顾或博客 API 时，才由用户或 AI 另建 `Sources/` source record；这个 Source 可以手动链接回 clipping。
- 没有后续用途的剪藏可以留在 `Clippings/`，不用强行补 Source 或 Daily 回链。
