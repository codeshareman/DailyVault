# Notes

闪念、个人思考、临时研究、非正式主题笔记。

规则：

- 闪念先快速记下来，不要求完整结构。
- 可以粗糙，允许半成品。
- Daily 只链接当天相关笔记，不复制全文。
- 每日复盘时处理：放弃、补充、链接到项目，或转成正式笔记。
- 当一条笔记变成稳定项目、文章、学习材料或长期研究时，再迁移到对应工程。

## Dataview 友好字段

Notes 用来承接闪念，不要求完整结构；但 frontmatter 至少应保留：

- `note_type: fleeting-note`
- `status: inbox | clarified | linked | archived`
- `related_daily`
- `visibility`

统计时重点看：

- 各状态 Notes 数量；
- 仍待处理的 `note-triage` 任务；
- 哪些 Notes 已链接回 Daily 或 Source。

不要把 Notes 当作正式知识库；稳定项目、文章、学习材料或长期研究应迁移到对应工程。
